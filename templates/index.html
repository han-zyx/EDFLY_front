About

EDFLY is a cutting-edge edge-to-cloud data streaming and monitoring solution designed to enhance real-time data processing in distributed environments. By integrating Apache Flink for stream processing, YOLOv8 for object detection, and Prometheus-Grafana for monitoring, EDFLY ensures seamless and low-latency data flow between edge devices and the cloud. This system empowers businesses and researchers with real-time analytics, making it ideal for smart surveillance, IoT applications, and industrial automation.

Why?

With the growing demand for real-time data processing, traditional cloud-based architectures struggle with high latency, bandwidth limitations, and reliability issues. EDFLY addresses these challenges by leveraging edge computing, reducing dependency on centralized servers, and enabling faster decision-making. It is designed for scalability, allowing multiple edge devices to process and stream data efficiently while ensuring data accuracy and security.

Research Gap

Existing real-time data processing solutions primarily focus on cloud-based architectures, which often introduce latency and network dependency issues. While edge computing offers a solution, there is a lack of integrated frameworks that combine stream processing, real-time object detection, and efficient monitoring in a single system. Additionally, many current approaches struggle with seamless integration between edge devices and cloud platforms, leading to inefficiencies in data management and decision-making.
